# Code of Conduct

## Our pledge

We want GrowthLens to be a welcoming, harassment-free space for everyone —
regardless of experience level, background, or whether you write code for a
living. Most contributors here are marketers, analysts, and growth people, not
software engineers. That is exactly who this project is for.

## Our standards

Examples of behavior that helps the community:

- Being respectful of differing viewpoints and experience levels.
- Giving and gracefully accepting constructive feedback on skills and reports.
- Focusing on what is best for the people who actually use these skills.
- Sharing real methodology and crediting sources.

Examples of unacceptable behavior:

- Harassment, insults, or derogatory comments.
- Publishing others' private information without permission.
- Submitting skills that scrape or process data in ways that violate a target
  site's terms of service, or that target individuals rather than businesses.
- Spam, plagiarism, or passing off others' frameworks as your own.

## Scope

This Code of Conduct applies in all project spaces — issues, pull requests,
discussions — and when representing the project in public.

## Enforcement

Report unacceptable behavior by opening a confidential issue or contacting the
maintainer through their GitHub profile (`@fjilvi-afk`). Maintainers may remove
comments, commits, issues, and contributions that violate this Code of Conduct,
and may temporarily or permanently ban any contributor for behaviors they deem
inappropriate.

This Code of Conduct is adapted from the [Contributor Covenant](https://www.contributor-covenant.org),
version 2.1.
